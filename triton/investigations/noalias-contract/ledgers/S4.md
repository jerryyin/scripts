## HOW-facts — each hop proven on real IR/asm

### Hop 0 — EMISSION: `noalias` on the GatherIndx arg (pre-opt LLVM), and nothing else differs
Pre-opt `define` (`LLVM_IR_ENABLE_DUMP`, first occurrence, line 1699252 of the dump). `%19` is
the 8th pointer arg = GatherIndx (S1 mapping). The ONLY difference between the two variants:
```
AFTER : ... ptr addrspace(1) noalias %19, ptr addrspace(1) %20, ...
BEFORE: ... ptr addrspace(1)         %19, ptr addrspace(1) %20, ...
```
Every other pointer arg is bare in both (matches S1). This is the kill switch at emission.

### Hop 1 — INFERENCE: `FunctionAttrs` folds noalias into `readonly captures(none)` — in BOTH
Post-opt `.llir` define, `%19` (GatherIndx):
```
AFTER : ptr addrspace(1) noalias nofree readonly captures(none) %19
BEFORE: ptr addrspace(1)         nofree readonly captures(none) %19
```
**Key refinement of the naive chain:** `FunctionAttrs` infers `readonly captures(none)` on `%19`
*even in BEFORE* (the kernel never writes through it). So `readonly` is present in BOTH states,
yet BEFORE still has 16 in-loop rfl. **`readonly` is therefore neither the differentiator nor a
necessary step for the win.** `FunctionAttrs` runs regardless; `noalias` is the arg attr it does
NOT synthesize and cannot be derived — it must be emitted by the contract.
Full-file diff of the two post-opt `.llir` = **exactly one token**: `noalias` on `%19`. All 325
`load` ops and every body line are byte-identical. This is the cleanest A/not-A isolation:
one attribute, same stock `llc`, opposite asm.

### Hop 2 — the LLVM-IR load is identical & carries NO `!invariant.load`
The gather load is `%187 = load <1 x i16>, ptr addrspace(1) %124, align 2, !dbg !40` (and 31
siblings), **identical** in BEFORE and AFTER. `grep !invariant.load` = **0** in both modules.
=> Load invariance is NOT expressed as IR metadata and NOT what changes. The divergence is
entirely inside the AMDGPU backend, driven by the `noalias` param attr via alias analysis.

### Hop 3 — the exact AMDGPU pass and predicate (named, not paraphrased)
`llvm/lib/Target/AMDGPU/AMDGPUAnnotateUniformValues.cpp:67 visitLoadInst`:
```
69: if (UA->isDivergentAtDef(Ptr)) return;      // uniformity leg
73: setUniformMetadata(PtrI);                    // -> !amdgpu.uniform
80: bool GlobalLoad = (addrspace == GLOBAL);
81: if (GlobalLoad && !AMDGPU::isClobberedInFunction(&I, MSSA, AA))
82:   setNoClobberMetadata(&I);                   // -> !amdgpu.noclobber
```
`isClobberedInFunction` (`AMDGPUMemoryUtils.cpp:409`) walks MemorySSA clobbers and at line 437
calls `isReallyAClobber(... Def, AA)` -> `AA->alias(load, store)`. With `noalias` on the base
pointer, every candidate store (the writeonly outputs %0/%2/%6/%14) returns **NoAlias** ->
no clobber found -> returns false -> `!amdgpu.noclobber` set. Without `noalias`, the i16 gather
load *may-alias* those stores -> `isReallyAClobber` true -> **not** annotated. The scalar
selection (`SIISelLowering`, gated by `-amdgpu-scalarize-global-loads`, default on) fires only
for a uniform + noclobber global load. `noalias` supplies the noclobber leg; the uniform leg is
satisfied in both variants (see Hop 4).

### Hop 4 — the asm consequence, both directions (16 -> 0) and the descriptor hoist
Prologue index load (stock `llc`, default):
```
BEFORE: global_load_u16 v1, v3, s[8:9] scale_offset   (x32, into VGPRs)
AFTER : s_load_u16      s5, s[8:9], 0x0                (x32, into SGPRs)
```
In BEFORE the loaded indices are assembled into the TDM `tensor_load_to_lds` descriptor in
VGPRs (v52..v67, built once in the prologue lines 828..1007 -> loop-invariant). Because they sit
in VGPRs, the hot loop re-lifts them to SGPR **every K iteration**: 16x `v_readfirstlane_b32
s56..s71, v52..v67` at hot-loop lines 1423..1443, feeding `tensor_load_to_lds ... s[56:59],
s[60:63]`. In AFTER the same descriptor is already in SGPRs (scalar `s_load`), so the loop body
needs zero re-lift. This is the "hoist": the value is materialized once in the prologue in the
right (scalar) register class instead of being recomputed/re-copied per iteration.
Measured hot loop: **BEFORE in_loop_rfl=16, AFTER in_loop_rfl=0** (prologue s_load_u16: 0 vs 32).

### Hop 5 — pin the pass with a 2x2 (proves noalias AND scalarization are each necessary)
Same two `.llir`, stock `llc`, toggling `-amdgpu-scalarize-global-loads`:
```
                       scalarize=ON(default)          scalarize=OFF
AFTER (noalias)   rfl=0,  s_load_u16=32          rfl=16, global_load_u16=32
BEFORE(no noalias)rfl=16, global_load_u16=32     rfl=16, global_load_u16=32
```
`s_load`/rfl=0 appears IFF `noalias` present AND scalarization on. BEFORE+scalarize = still
vector (pass runs but AA can't prove noclobber); AFTER+noscal = still vector (attr present but
consumer disabled). Clean double dissociation.

## CLAIM STATUS

**CLAIM (answered, irrefutable at IR/asm level):** The canonical F1-prefill mechanism is
`noalias` (emitted, arg %19) -> [FunctionAttrs folds to `noalias nofree readonly captures(none)`;
readonly is incidental] -> AMDGPU `AMDGPUAnnotateUniformValues.visitLoadInst`: uniform pointer +
`!isClobberedInFunction` (AA(noalias)=NoAlias vs writeonly stores) => `!amdgpu.noclobber` -> the
global-load scalarizer selects `s_load_u16` -> the TDM descriptor is built in SGPRs in the
prologue -> the 16 per-iteration `v_readfirstlane` vanish (16->0).

Necessary / sufficient / neither:
- **noalias emission:** NECESSARY. (BEFORE with everything else identical => rfl=16.) Sufficient
  ONLY jointly with the default-on `-amdgpu-scalarize-global-loads` pass (AFTER+noscal => 16).
- **readonly inference:** NEITHER necessary nor sufficient. Present in both variants; win tracks
  noalias, not readonly.
- **invariance:** the operative form is backend `!amdgpu.noclobber` (AA-derived from noalias),
  NOT LLVM-IR `!invariant.load` (absent) and NOT `readonly`. The uniformity leg is a separate,
  noalias-independent precondition (satisfied in both variants — BEFORE's rfl prove the value is
  uniform, merely stranded in VGPR).

**Counter-experiments that would refute (and their results):**
1. "Remove only noalias, keep all else, and still get s_load/rfl=0." -> BEFORE: global_load,
   rfl=16. Not refuted.
2. "Show readonly is the switch (present only in AFTER)." -> readonly present in BOTH. Not refuted.
3. "Disable scalarization on the noalias IR and still get rfl=0." -> AFTER+noscal: rfl=16.
   Confirms joint necessity; not refuted.
4. "Find `!invariant.load` metadata carrying the invariance." -> 0 occurrences in both. The
   carrier is `!amdgpu.noclobber`; not refuted.

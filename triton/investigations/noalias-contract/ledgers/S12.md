### FACT 1 — F4 read-gather (`gather_kv_b_proj`): the index load is a DIVERGENT VECTOR buffer_load
Baseline (no noalias) gfx1250, arg `%3` = `kv_indices`:
```

`readonly` is inferred WITHOUT noalias (matches S3 Fact 1 / S4 Hop 1). In TTGIR the gather index is

```
i.e. 16 lanes each load a DIFFERENT `kv_indices` entry (`+ arange(0,ChunkK)//KBlockSize`) — a
**divergent, per-lane vector** load. Asm: it stays VMEM `buffer_load_*` (never `s_load`). By S5's

## Applying the S3/S4/S5 canon to F4 (why the mechanism cannot pay off here)

- **S4 canon (noalias ⇒ AA NoAlias ⇒ `!amdgpu.noclobber` ⇒ global-load scalarizer ⇒ s_load, killing
  per-iter re-lift):** requires a GLOBAL load that (i) is uniform and (ii) would otherwise be treated
  as clobbered by an in-function store, AND (iii) is reused across a loop. F4 has no case with all
  three: the read-gather fails (i) (divergent) and is a `buffer_load` not a scalarizer-eligible
  `global_load`; the scatter-index fails (iii) (no loop) and already needs no (ii).
- **S5 truth table:** `s_load` fires IFF uniform + (invariant|noclobber) + align≥4 + global/const +
  scalarize-on + representable-as-scalar-SMEM. The read-gather is DIVERGENT + VECTOR — two
  independent disqualifiers (S5 Fact F, Fact C). The scatter-index satisfies the table and indeed is
  `s_load` — but it reaches it in the BASELINE (noalias not the differentiator, as S5 Fact E's
  corollary predicts: `readonly` alone makes a single entry load invariant/noclobber when no store
  can clobber it).
- **S3 soundness (applied to any FUTURE F4 annotation):** were noalias ever added to a real F4 index
  arg, the S3 two-part contract must hold: (1) readonly — never written through the index pointer
  (true: `slot_mapping`/`kv_indices`/`kv_indptr` are read-only inputs, and `tl.const` is not even
  needed since FunctionAttrs infers readonly), and (2) noalias — the index buffer must not overlap
  any WRITABLE kernel arg. For `cat_and_cache_mla` the writable arg is `kv_cache_ptr` (arg %2,
  `writeonly`) — `slot_mapping` and `kv_cache` are distinct tensors, so the promise holds; BUT since
  Fact 4 shows zero codegen benefit, taking on the S3 silent-miscompile liability (stale-index CSE +
  gfx1250 K$ incoherence with the kv_cache vector store) buys nothing. NET: do not annotate F4.
